# 法務・到達率チェックリスト

物件情報メールを配信する前に、最低限以下を確認してください。

## 1. 配信許諾

- [ ] 受信者から広告・案内メールの送信許諾を得ている
- [ ] 許諾取得日を `opt_in_at` に記録している
- [ ] 許諾取得方法を `opt_in_method` に記録している
- [ ] 名刺交換、問い合わせ、既存取引先など、送信根拠を説明できる
- [ ] 購入・取得元が不明なリストへ送らない

参考:

- 消費者庁 特定商取引法ガイド「迷惑メールの概要」: https://www.no-trouble.caa.go.jp/spam/about.html
- 迷惑メール相談センター「特定電子メール法」: https://www.dekyo.or.jp/soudan/contents/taisaku/1-2.html

## 2. 表示義務・本文記載

本文末尾に最低限以下を入れてください。

- [ ] 送信者名または会社名
- [ ] 問い合わせ先メールアドレス
- [ ] 配信停止方法
- [ ] 住所、または事業者情報が分かるリンク
- [ ] 苦情・問い合わせ先

例:

```text
配信元: 株式会社XXXX
所在地: 東京都...
お問い合わせ: info@example.jp
配信停止: このメールに「停止」と返信してください。
```

## 3. 配信停止

- [ ] 停止希望を受けたら即 `status=stopped` にする
- [ ] 停止者へ再送しない
- [ ] 停止日時・停止理由を記録する
- [ ] 大量配信では配信サービスの解除リンクを使う
- [ ] 将来的にワンクリック解除に対応する

Google はメール送信者ガイドラインで、商用・購読系メールに登録解除対応を求めています。

参考:

- Google Email sender guidelines: https://support.google.com/a/answer/81126

## 4. 送信者認証

大量配信では独自ドメインを使い、以下を設定します。

- [ ] SPF
- [ ] DKIM
- [ ] DMARC
- [ ] Fromドメインと認証ドメインの整合
- [ ] `gmail.com` の個人アドレスを本番大量配信の送信元にしない

## 5. バウンス・苦情管理

- [ ] ハードバウンスは `status=bounced` にする
- [ ] ソフトバウンスは回数を記録する
- [ ] 迷惑メール報告は即停止する
- [ ] 古いリストを定期的に整理する
- [ ] 反応がない宛先をセグメントから外す

## 6. Gmail無料運用の上限

無料Gmailはテスト用途に限定してください。

- [ ] `TEST_MODE=true` でテストしている
- [ ] `MAX_SEND_PER_RUN` を小さくしている
- [ ] 100名以上に送る前に配信サービス移行を検討している
- [ ] Bcc一斉送信を使っていない

Google Apps Script のクォータでは、Consumer account の MailApp 受信者数は `100 / day` です。

参考:

- Google Apps Script quotas: https://developers.google.com/apps-script/guides/services/quotas

## 7. 送信前チェック

毎回送信前に確認します。

- [ ] テスト送信でリンクが正しく開く
- [ ] 価格、エリア、物件URLに誤りがない
- [ ] 配信停止方法が本文にある
- [ ] `status=active` かつ `consent=yes` だけが対象
- [ ] 停止者・バウンス者が除外されている
- [ ] 送信ログが残る

## 8. 大量配信前チェック

1,000名以上へ送る前に確認します。

- [ ] 独自ドメインを取得済み
- [ ] SPF/DKIM/DMARC設定済み
- [ ] 配信サービスを契約済み
- [ ] 配信停止リンクを用意済み
- [ ] バウンスWebhookを処理できる
- [ ] 迷惑メール報告を検知できる
- [ ] 古いリストをクリーニング済み
- [ ] いきなり全件送らず、小さなセグメントから送る

## 運用ルール例

```text
1日目: 自分・社内のみ
2日目: 10件
3日目: 30件
4日目: 50件
5日目以降: 反応・エラーを見ながら拡大
```

大量配信では、配信数を増やすことよりも、停止・エラー・苦情を正しく処理することを優先してください。
